# Анализ тональности сообщений ВКонтакте


Система для сбора, анализа и визуализации тональности сообщений из социальной сети «ВКонтакте» с использованием методов машинного обучения (TF‑IDF + логистическая регрессия, RuBERT) и LLM (Saiga) для тематического анализа.

## Основные возможности

- Сбор данных через официальный VK API (посты и комментарии) с кэшированием.
- Предобработка текста: очистка, нормализация эмодзи и смайлов, лемматизация (опционально).
- Анализ тональности двумя моделями:
  - Baseline: TF‑IDF + логистическая регрессия (быстрая, интерпретируемая).
  - RuBERT: предобученный трансформер (rubert-tiny) с возможностью дообучения на ваших данных.
- Визуализация:
  - Интерактивные графы сходства текстов (цвет по группе или тональности).
  - Иерархический граф «сообщество → теги → посты → комментарии».
- Анализ причин:
  - Извлечение ключевых n-грамм из негативных/позитивных комментариев.
  - (Опционально) LLM-анализ с помощью локальной модели Saiga (через Ollama) с учётом контекста поста.
- Валидация моделей на отложенной тестовой выборке с генерацией HTML-отчёта.
- Модульное тестирование (pytest) для функций предобработки.

## Установка и зависимости

### Требования

- Python 3.10 или выше
- Ollama (только если используется LLM-анализ)
- Токен доступа к VK API

### Установка

1. Клонируйте репозиторий:

   ```bash
   git clone https://github.com/your-repo/vk-sentiment.git
   cd vk-sentiment
   ```
2. Создайте виртуальное окружение:

   ```bash
   python -m venv venv
   source venv/bin/activate      # Linux/Mac
   venv\Scripts\activate         # Windows
   ```
3. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```
4. Настройте токен VK: создайте файл `.env` в корне проекта:

   ```
   VK_TOKEN=ваш_токен_здесь
   ```
5. (Опционально) Установите и запустите Ollama:

   ```bash
   curl -fsSL https://ollama.com/install.sh | sh
   ollama pull Double00/saiga_llama3:latest
   ```

## Запуск

### 1. Сбор данных и анализ (основной режим)

```bash
python main.py
```

После выполнения в папке `output/` появятся:

- `vk_sentiment_results.csv` – таблица с предсказаниями обеих моделей.
- `graph_groups.html`, `graph_sentiment_rubert.html` – интерактивные графы.
- `hierarchy_graph.html` – иерархический граф.
- `reasons_report.html` – отчёт по n-граммам.
- `saiga_reasons_report.html` – LLM-отчёт (если `USE_LLM = True`).

### 2. Валидация моделей (оценка на отложенной тестовой выборке)

```bash
python main.py --validate
```

Создаст `validation_report.html` с метриками accuracy, macro/weighted F1, матрицами ошибок.

### 3. Только сбор и анализ (без графов)

```bash
python main.py --no-graphs
```

### 4. Ручная разметка данных (для дообучения RuBERT)

- Подготовка выборки: `python prepare_labeling.py` (создаст `to_label.csv`)
- Разметка тональности: `python labeling_tool.py` (интерактивно вводите 0/1/2 или q для выхода). Результат сохраняется в `labeled_data.csv`.

### 5. Дообучение RuBERT на размеченных данных

```bash
python finetune_rubert.py
```

Обученная модель сохранится в `models/fine_tuned_model/`. После этого `main.py` будет использовать дообученную версию.

### 6. Запуск тестов

```bash
pytest tests/ -v
```

## Описание компонентов


| Компонент                  | Тип                                                  | Вход                                        | Выход                                                                                  | Назначение                                                                                                                              |
| ----------------------------------- | ------------------------------------------------------- | ----------------------------------------------- | ------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| Baseline                            | TF‑IDF + логистическая регрессия | Текст (лемматизированный) | negative/neutral/positive                                                                   | Быстрая, интерпретируемая классификация (бейзлайн)                                                    |
| RuBERT                              | Трансформер (rubert-tiny)                    | Исходный текст                     | (sentiment, confidence)                                                                     | Точная классификация с учётом контекста                                                                        |
| Saiga                               | LLM (Ollama)                                            | Пара (пост, комментарий)     | Список тем, частота, примеры, резюме, рекомендации | Качественный тематический анализ и бизнес-выводы                                                       |

| Иерархический граф | networkx + plotly                                       | DataFrame                                       | HTML-граф                                                                               | Отображение связей сообщество → теги → пост → комментарии (цвет по тональности) |

## Валидация и качество моделей

После разметки не менее 500 текстов выполните `python main.py --validate`. Пример отчёта:


| Модель        | Accuracy | Macro F1 | F1 (негатив) | F1 (нейтраль) | F1 (позитив) |
| ------------------- | -------- | -------- | ------------------- | --------------------- | ------------------- |
| Baseline            | 0.72     | 0.71     | 0.70                | 0.68                  | 0.74                |
| RuBERT (fine-tuned) | 0.89     | 0.88     | 0.87                | 0.86                  | 0.91                |

*Результаты могут отличаться в зависимости от размера и качества разметки.*

## Конфигурация

Все настройки находятся в `config.py`:

- Список сообществ (`GROUPS`)
- Количество постов (`POSTS_PER_GROUP`)
- Пути к моделям и выходным папкам (`OUTPUT_DIR`, `MODELS_DIR`, `LOGS_DIR`)
- Включение/отключение LLM (`USE_LLM`)
- Параметры графа (`SIMILARITY_THRESHOLD`, `TFIDF_MAX_FEATURES`)

## Тестирование

Тесты покрывают функции очистки, нормализации эмодзи и лемматизации:

```bash
pytest tests/ --cov=preprocessing
```

## Часто задаваемые вопросы

**1. Что делать, если VK API возвращает ошибку 5 (авторизация)?**
Проверьте, что в `.env` указан актуальный токен доступа и он не истёк.

**2. Почему таблицы в LLM-отчёте пустые или сломаны?**
Удалите файл `output/saiga_cache.json` и перезапустите `main.py`. Убедитесь, что Ollama запущена.

**3. Как ускорить работу RuBERT?**
Используйте GPU (установите `torch` с поддержкой CUDA) или уменьшите `batch_size` в `sentiment_models.py`.

**4. Где должен лежать `labeled_data.csv`?**
В корне проекта (или в папке `data/`, если изменили путь в `finetune_rubert.py`).
